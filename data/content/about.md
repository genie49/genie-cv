# 김형진 | AI Engineer

한번 몰입하면 빠르게 결과물을 만들어내는 개발자입니다. AI 기술이 실제 비즈니스 가치를 만드는 과정에 강한 흥미를 가지고 있으며, 딥러닝 모델부터 LLM 기반 Multi-Agent 시스템, 풀스택 웹 서비스까지 아이디어를 프로덕트로 완성하는 전 과정을 경험했습니다.

## 핵심 역량

### 멀티에이전트 시스템 설계

복잡한 비즈니스 워크플로우를 에이전트 협업 구조로 풀어온 경험이 있습니다.

- **fingoo**: Supervisor 패턴으로 시장분석·기술분석·리서치·퀀트 등 7개 전문 에이전트를 오케스트레이션하는 투자 분석 시스템을 설계했습니다. 81개 도구를 에이전트별로 분배하고, 10개 미들웨어 스택으로 컨텍스트를 격리하여 에이전트 간 간섭 없이 병렬 실행이 가능하도록 구현했습니다.
- **kimpro**: Account Manager가 5개 전문 에이전트에게 캠페인 기획 → 브리프 → 매칭 → 가이드 4단계 워크플로우를 위임하는 구조를 설계했습니다. Redis CAS 패턴으로 각 단계의 실행 결과를 스냅샷하고, 실패 시 원자적 롤백이 가능하도록 구현하여 프로덕션 안정성을 확보했습니다.
- **hey-bara**: 안드로이드 온디바이스 환경에서 Koog 프레임워크 기반 AI Agent가 14개 도구(전화·문자·일정·앱 제어)를 Function Calling으로 실행하는 음성 비서를 구현했습니다. Human-in-the-Loop 확인 패턴으로 위험 액션에 대한 사용자 승인 절차를 설계했습니다.

### 컨텍스트 엔지니어링 & RAG 파이프라인

에이전트가 올바른 판단을 하려면 적절한 컨텍스트가 필수입니다. 다양한 데이터 소스와 규모에 맞는 RAG 파이프라인을 설계해왔습니다.

- **bonda**: 한국 고고학 발굴보고서 PDF를 대상으로 PyMuPDF → PaddleOCR 폴백 → 2000자 청킹 → Gemini Embedding → Qdrant 3개 컬렉션 저장까지의 End-to-End RAG 파이프라인을 구축했습니다. 텍스트뿐 아니라 보고서 내 유물 사진을 자동 크롭하고, 듀얼 벡터(이미지 + 텍스트 설명)로 임베딩하여 텍스트 검색과 시각적 유사도 검색을 동시에 지원합니다.
- **fingoo**: pgvector 기반 1차 검색 → 컨텍스트 충분성 판단 → 불충분 시 Tavily 웹 검색으로 보완하는 2단계 RAG를 설계했습니다. CSV·PDF·이미지 등 멀티모달 입력을 AI가 이해할 수 있는 형태로 변환하는 파이프라인을 구현했습니다.
- **genie-cv**: LanceDB + Gemini Embedding으로 포트폴리오 콘텐츠를 H2 헤더 기반으로 청킹하고, 소스 인용 시스템으로 답변 신뢰성을 확보하는 경량 RAG 챗봇을 구축했습니다.

### Agent 품질 평가 체계

프롬프트를 수정할 때마다 다른 곳이 망가지는 문제를 겪으면서, 에이전트 품질을 체계적으로 관리하는 방법을 고민해왔습니다.

- **fingoo**: MockService 기반 평가 환경을 구축하여 실제 API 호출 없이 에이전트 동작을 검증할 수 있도록 했습니다. 정확성·완전성·관련성·안전성·형식 준수·도구 사용 6가지 Grader로 에이전트 품질을 다각도로 수치화하고, 평가 결과를 기반으로 프롬프트를 자동 개선하는 피드백 루프를 만들었습니다.
- **kimpro**: Anthropic Prompt Caching을 적용하여 반복 시스템 프롬프트의 비용과 지연 시간을 최적화하고, Agent 품질 평가 체계를 구축하여 프롬프트 변경 시 회귀를 방지했습니다.

### 비즈니스 문제를 Agent 워크플로우로 전환

기술 중심이 아니라 비즈니스 병목을 먼저 파악하고, 그에 맞는 Agent 구조를 설계하는 접근 방식을 지향합니다.

- **kimpro**: 마케팅팀이 캠페인 하나를 진행하는 데 수동으로 거치던 기획 → 브리프 → 인플루언서 매칭 → 가이드 생성 프로세스를 분석했습니다. 각 단계를 Agent 노드로 변환하여 4단계 자동화 워크플로우를 구축하고, GCP Pub/Sub 기반 이벤트 드리븐 마이크로서비스로 7개 서비스(Auth·API·Chat·Workflow·Tools·Notification·History)를 안정적으로 운영했습니다.
- **bonda**: 문화재 연구자들이 발굴보고서 PDF를 일일이 열어 검색해야 했던 문제를 파악하고, 크롤링 → 파싱 → 벡터화 → 시맨틱 검색까지의 자동화 파이프라인으로 해결했습니다. 13,000개 이상의 고고학 용어 사전을 구축하여 도메인 특화 검색 품질을 높였습니다.

### 풀스택 E2E 구현력

프론트엔드부터 백엔드, AI, 모바일, 인프라까지 아이디어를 프로덕트로 완성해왔습니다.

- **프론트엔드**: React, Next.js, Jetpack Compose — 채팅 UI, 인터랙티브 차트, 모바일 앱
- **백엔드**: FastAPI, Fastify, ElysiaJS, NestJS — REST API, WebSocket, SSE 스트리밍
- **AI/ML**: LangChain, LangGraph, vLLM, PyTorch, Sherpa-ONNX — 멀티에이전트, RAG, ML 서빙, 온디바이스 STT/TTS
- **인프라**: Docker, AWS, GCP, Qdrant, PostgreSQL, MongoDB, Redis — 벡터 DB, 이벤트 드리븐 아키텍처
- 특히 Agent 시스템에 필수적인 실시간 통신(WebSocket, SSE)과 스트리밍 파이프라인을 여러 프로젝트에서 반복 구현하며 패턴을 체득했습니다.
