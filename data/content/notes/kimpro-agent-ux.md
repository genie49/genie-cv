# 말많은 수다쟁이를 멋들어진 발표자로

AI에게 "캠페인을 기획해줘"라고 요청하면 3분 뒤 결과가 돌아온다. 그 3분 동안 사용자가 보는 건 스피너 하나. 30초쯤 지나면 "작동하는 건 맞아?"라는 불안이 시작되고, 1분이 넘으면 새로고침을 누른다.

3분 뒤 결과가 한꺼번에 쏟아지면 그건 **정보의 나열**이지 답변이 아니다. 마치 3분간 침묵하다 한꺼번에 쏟아내는 수다쟁이. 이걸 "지금 어디까지 했고, 무엇을 하고 있고, 무엇이 바뀌었는지"를 실시간으로 보여주는 발표자로 바꾸는 과정을 정리한다.

## AI의 행동을 세 가지 층으로 나눈다

에이전트가 일하는 동안 사용자에게 전달해야 할 정보는 세 가지 추상 수준으로 나뉜다.

| 층 | 무엇을 알려주는가 | 사용자가 보는 것 |
|---|---|---|
| 텍스트 | 지금 무슨 말을 하고 있는지 | 채팅 버블에 글자가 한 글자씩 나타남 |
| 행동 | 지금 무슨 일을 하고 있는지 | "제품 분석 중", "크리에이터 매칭 시작" |
| 변화 | 데이터가 어떻게 바뀌었는지 | 옆 패널의 캠페인 정보가 실시간으로 갱신 |

**텍스트 스트리밍**은 LLM의 토큰 단위 출력을 그대로 채팅 버블에 흘려보낸다. 글자가 하나씩 나타나는 타이핑 효과. 이것만으로도 "AI가 지금 말하고 있다"는 느낌을 준다. 스피너보다 훨씬 강력한 피드백이다.

**행동 감지**는 에이전트가 도구를 호출할 때 발생한다. 서브에이전트를 호출하면 "제품 분석 시작", 도구 실행이 끝나면 "제품 분석 완료". 사용자는 AI가 **뭔가를 하고 있다**는 걸 인지한다.

**데이터 변화**는 에이전트가 캠페인 데이터를 수정할 때마다 실시간으로 반영된다. 에이전트가 제품명을 수정하면 옆 패널에서 바로 바뀐다. 채팅이 끝나면 **변경사항 요약 카드**가 나타나서, 이번 대화에서 뭐가 달라졌는지 한눈에 볼 수 있다.

세 가지 층을 조합하면 사용자는 AI의 작업을 **입체적으로** 경험한다. 말만 하는 게 아니라, 행동하고, 결과를 남긴다.

## TODO로 발표 구조를 만든다

수다쟁이와 발표자의 가장 큰 차이는 **구조**다. 발표자는 "오늘 이 세 가지를 다루겠습니다"라고 먼저 말한다.

에이전트도 마찬가지다. 작업을 시작하면 먼저 **TODO 리스트를 생성**한다. "제품 분석 → 타겟 오디언스 정의 → 전략 수립 → 크리에이터 매칭" 같은 단계별 목록이 사용자에게 보인다. 에이전트가 각 단계를 완료할 때마다 TODO가 체크된다.

이 패턴의 효과는 두 가지다.

첫째, **체감 대기시간이 줄어든다.** 3분이 걸리더라도 "4단계 중 2단계 완료"를 보고 있으면 기다릴 수 있다. 진행률이 보이는 3분과 암흑 속의 3분은 체감이 완전히 다르다.

둘째, **AI의 사고 과정이 투명해진다.** 사용자는 에이전트가 어떤 순서로 문제를 풀어가는지 볼 수 있다. "아, 제품부터 분석하고 그다음 타겟을 정하는구나." 결과만 받아보는 것과 과정을 지켜보는 것은 신뢰도가 다르다.

## 한 명의 발표자, 다섯 명의 전문가

뒤에서는 서브에이전트 5개가 동시에 일한다. 제품 분석가, 전략가, 콘텐츠 기획자, 매칭 전문가 — 각각 자기 영역에서 도구를 호출하고 데이터를 수정한다.

그런데 이들의 텍스트가 전부 채팅에 뜨면 어떻게 될까. 5명이 동시에 떠들기 시작한다. "제품 카테고리는 뷰티입니다", "타겟 연령대를 분석하고 있습니다", "크리에이터 풀을 검색합니다" — 사용자는 **누가 뭘 하는 건지** 혼란스러워진다.

그래서 **Supervisor의 텍스트만 채팅에 표시**한다. 서브에이전트의 출력은 Supervisor에게만 전달되고, Supervisor가 이를 종합해서 사용자에게 전한다. 사용자 입장에서는 **한 명의 AI 비서**와 대화하는 느낌. 뒤에서 전문가 팀이 일하고, 앞에서 한 명이 정리해서 발표하는 구조다.

서브에이전트의 **행동**은 보여준다. "제품 분석 시작", "매칭 완료" 같은 상태는 사용자에게 전달된다. 말은 한 명이 하지만, 팀이 일하고 있다는 건 알 수 있다. 이 균형이 중요하다 — 투명하되 혼란스럽지 않게.

## 중단할 수 있어야 대화다

발표자가 일방적으로 3분간 말하면 그건 독백이다. 사용자가 언제든 "잠깐, 방향을 바꿀게"라고 말할 수 있어야 **대화**가 된다.

에이전트가 도구를 호출하는 중이라 해도 사용자의 중단 요청을 즉시 처리한다. 도구 호출이 30초째 실행 중인 상황에서도, 중단 시그널이 들어오면 현재 작업을 멈추고 그때까지의 부분 결과를 저장한다. "5개 서브에이전트 중 3개가 끝났고 2개가 진행 중이었다면, 3개의 결과라도 보존"하는 식이다.

이건 기술적으로는 스트리밍 루프에서 중단 시그널과 다음 청크를 경합시키는 패턴으로 구현된다. 하지만 본질은 기술이 아니라 UX다. **사용자가 주도권을 쥐고 있다는 느낌**. AI가 일방적으로 처리하고 결과를 통보하는 게 아니라, 사용자가 흐름을 제어할 수 있다.

## 돌이켜보면

결국 만든 것은 **AI의 일하는 과정을 사용자 경험으로 번역하는 층**이었다.

토큰은 타이핑 효과로, 도구 호출은 행동 알림으로, 데이터 수정은 실시간 반영으로, 작업 계획은 TODO 리스트로, 서브에이전트는 하나의 목소리로. 같은 에이전트라도 이 번역 층이 있고 없고에 따라 사용자가 느끼는 품질이 **완전히 달라진다.** 모델을 바꾸지 않아도, 보여주는 방식을 바꾸면 체감 성능이 바뀐다.
