# QnA 섹션 리디자인 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** QnA 섹션을 12개 범용 질문에서 7개 STAR 기반 에피소드 중심으로 전면 교체하여, 범용성을 유지하면서 토스 핵심가치(임팩트, 빠른 실행, 피드백 문화)를 자연스럽게 녹여냄.

**Architecture:** `data/qna.json` 데이터만 전면 교체. UI 컴포넌트(`QnAPage.tsx`)와 타입(`QnAItem`)은 변경 없음. 교체 후 `scripts/embed.ts`를 재실행하여 RAG 임베딩 갱신.

**Tech Stack:** JSON, TypeScript (embed script)

**Spec:** `docs/superpowers/specs/2026-03-26-qna-redesign-design.md`

## 답변 작성 가이드 (모든 Task 공통)

각 답변은 다음 규칙을 따른다:

- **분량**: 400~600자 (자기소개만 예외 허용)
- **구조**: 상황 → 판단 → 행동 → 결과/배움 (STAR 라벨은 명시하지 않음)
- **첫 문장**: 핵심 메시지를 담을 것
- **프로젝트**: 메인 1개 + 보조 최대 1개
- **마무리**: 짧고 임팩트 있는 한 문장
- **톤**: Self Q&A 자문자답, "~입니다/~했습니다" 체
- **JSON 형식**: `{"question": "질문", "answer": "답변"}` — `QnAItem` 타입 (정의: `packages/shared/src/index.ts`)

**작업 방식**: Task 1~7은 답변 방향을 정의한다. 실제 JSON 파일 작성은 **Task 8에서 한 번에** 수행한다. Task 1~7에서는 `data/qna.json`을 직접 수정하지 않는다.
