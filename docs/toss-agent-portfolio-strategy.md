| 컨텍스트 엔지니어링 | ✅ fingoo(RAG 2단계 파이프라인), genie-cv(LanceDB RAG) | 컨텍스트 윈도우 관리 전략 명시적으로 서술 |
| Agent 메모리 시스템(대화/세션/사용자 수준) 설계 운영 | ⚠️ fingoo(langchain-checkpoint), 부분적 경험 | **보강 권장**: 메모리 계층 설계 경험 추가 |
| Agent 품질 평가 체계 설계 운영 | ✅ fingoo(eval-harness, 6가지 Grader, MockService) | 이미 강점 — 수치와 개선 과정 강조 |
| 내부 팀 니즈 발굴 및 기술 스펙 번역 | ✅ kimpro(마케팅팀 요구 → Agent 워크플로우) | 구체적 사례 서술 |
| 반복 업무 AI 자동화 및 사용자 채택 | ✅ kimpro(인플루언서 마케팅 자동화) | 자동화 전후 비교 수치 추가 |
| 오픈소스 기여 | ❌ 확인 안됨 | **보강 권장**: 사용 중인 라이브러리에 PR/이슈 기여 |

## 2. 포트폴리오에서 보여줘야 할 5가지 핵심 역량

### 역량 1: 멀티에이전트 시스템 설계 능력

**왜 중요한가:** Agent 포지션의 핵심. 토스는 "Agent 실행 인프라 설계 및 운영"을 명시적으로 요구.

**보여줄 프로젝트:** fingoo + kimpro

**어떻게 보여줄 것인가:**
- fingoo의 Supervisor 패턴: 7개 전문 에이전트(시장분석·기술분석·리서치·퀀트 등)를 하나의 Supervisor가 오케스트레이션하는 구조
  - 왜 Supervisor 패턴을 선택했는가 (vs Swarm, Sequential)
  - 에이전트 간 상태 공유 방식
  - Human-in-the-Loop으로 사용자 확인 후 차트 조작하는 패턴
- kimpro의 순차 워크플로우: 4단계 캠페인 자동화를 LangChain 노드로 설계
  - 각 단계의 실패 처리와 재시도 전략
  - WebSocket으로 실시간 진행 상황 전달

**포트폴리오 노트 활용:**
- `fingoo-agentic-ai`: Supervisor·Subagent 패턴 설계 과정
- `fingoo-human-in-loop`: Human-in-the-Loop 차트 제어

---

### 역량 2: 컨텍스트 엔지니어링 & RAG 파이프라인

**왜 중요한가:** Agent가 올바른 판단을 하려면 적절한 컨텍스트가 필수. 토스 우대사항에 "컨텍스트 엔지니어링"이 명시됨.

**보여줄 프로젝트:** fingoo + genie-cv

**어떻게 보여줄 것인가:**
- fingoo의 2단계 RAG: 크롤링 → pgvector 검색 → 불충분 시 Tavily 웹 검색 보완
  - 컨텍스트 충분성 판단 로직
  - 임베딩 전략과 chunking 방식
- genie-cv의 경량 RAG: LanceDB + Gemini Embedding
  - 파일 기반 벡터 DB로 서버리스 운영
  - 소스 인용 시스템으로 답변 신뢰성 확보
- 멀티모달 컨텍스트: CSV·PDF·이미지를 AI가 이해하도록 변환하는 파이프라인

**포트폴리오 노트 활용:**
- `fingoo-rag-pipeline`: 2단계 RAG 파이프라인
- `fingoo-multimodal`: 멀티모달 입력 처리
- `rag-pipeline` (genie-cv): LanceDB RAG 설계

---

### 역량 3: Agent 품질 평가 체계

**왜 중요한가:** 토스 우대사항 "Agent 품질 평가 체계 설계 운영". Agent를 프로덕션에 배포하려면 품질 보장이 필수.

**보여줄 프로젝트:** fingoo

**어떻게 보여줄 것인가:**
- MockService 기반 평가 환경: 실제 API 호출 없이 에이전트 동작 검증
- 6가지 Grader: 다각도에서 에이전트 품질 측정
- 자가 피드백 기반 프롬프트 개선 파이프라인: 평가 결과 → 프롬프트 자동 개선 루프
- 평가 지표와 개선 수치를 구체적으로 제시

**포트폴리오 노트 활용:**
- `fingoo-eval-harness`: 평가 Harness 구축과 프롬프트 파이프라인

---

### 역량 4: 비즈니스 문제 → Agent 워크플로우 전환 능력

**왜 중요한가:** "사내 팀과의 협업을 통한 병목 발굴 및 기술 요구사항 전환"이 주요 업무.

**보여줄 프로젝트:** kimpro-agent-service

**어떻게 보여줄 것인가:**
- 마케팅팀의 반복 업무(캠페인 기획 → 브리프 작성 → 인플루언서 매칭 → 가이드 생성)를 4단계 Agent 워크플로우로 구조화한 과정
  - 기존 수동 프로세스 분석
  - 각 단계를 Agent 노드로 변환한 의사결정 근거
  - 자동화 전후 시간/비용 절감 수치
- 마이크로서비스 아키텍처: Auth, Chat, API, Workflow, Notification 등 서비스 분리 설계
- GCP 인프라: Cloud Storage, Secret Manager, Pub/Sub 활용

---

### 역량 5: 풀스택으로 E2E 구현하는 실행력

**왜 중요한가:** Agent 포지션은 "개발자 경험 및 공통 인프라 설계"도 담당. 내부 시스템과 LLM 통합 레이어를 직접 구축해야 함.

**보여줄 프로젝트:** 전체 프로젝트 포트폴리오

**어떻게 보여줄 것인가:**
- 프론트엔드(Next.js/React) + 백엔드(FastAPI/Fastify/ElysiaJS) + AI(LangChain) + 인프라(Docker/AWS/GCP)를 혼자 또는 소규모 팀으로 구축
- 아이디어 → 설계 → 구현 → 배포 → 운영까지 전체 사이클 경험
- 특히 Agent 시스템을 위한 실시간 통신(WebSocket, SSE) 구현 경험

## 3. 포트폴리오 사이트(genie-cv) 개선 방향

### 3.1 현재 상태

- 2개 프로젝트(fingoo, ai-portfolio-chatbot) 등록
- 프로젝트별 기능 설명 + 기술 노트 구조
- RAG 챗봇으로 방문자가 질문 가능

### 3.2 Agent 포지션 타겟 개선안

#### A. 프로젝트 추가 (우선순위 순)

1. **kimpro-agent-service** (필수 추가)
   - Agent 포지션 핵심 어필 프로젝트
   - 노트 예시:
     - "마케팅 자동화 Agent 워크플로우 설계 — 수동 프로세스에서 4단계 AI 파이프라인으로"
     - "LangChain 기반 마이크로서비스 오케스트레이션"
     - "GCP Pub/Sub + WebSocket 실시간 워크플로우 상태 전달"

2. **hey-bara** (추가 권장)
   - 온디바이스/클라우드 듀얼 모드 AI 설계 판단력을 보여줌
   - 노트 예시:
     - "온디바이스 vs 클라우드 LLM — 듀얼 모드 아키텍처 설계 의사결정"
     - "STT/TTS + LLM Agent 통합 파이프라인"

3. **influencer-classification** (선택)
   - ML 모델 학습 경험 보여주지만, Agent 포지션과 직접 연관은 낮음
   - 추가한다면 "딥러닝 모델부터 Agent까지" 스펙트럼을 보여주는 용도

#### B. 기술 노트 보강 (기존 프로젝트)

fingoo에 이미 좋은 노트가 있으나, Agent 포지션 관점에서 다음을 보강:

| 기존 노트 | 보강 포인트 |
|---|---|
| `fingoo-agentic-ai` | Supervisor vs Swarm vs Sequential 비교 판단 근거를 더 상세히 |
| `fingoo-eval-harness` | 6가지 Grader의 구체적 지표와 개선 수치 추가 |
| `fingoo-rag-pipeline` | "컨텍스트 엔지니어링" 관점으로 프레이밍 — 컨텍스트 윈도우 예산 관리 전략 |

#### C. 새로운 노트 추가 (Agent 관점)

1. **"Agent 메모리 아키텍처 — 대화/세션/사용자 수준 설계"**
   - fingoo나 kimpro에서의 상태 관리 경험을 메모리 시스템 관점으로 재구성
   - langchain-checkpoint 활용, 세션 영속성, 사용자별 컨텍스트 분리

2. **"Agent 장애 대응과 안전 장치 — 가드레일부터 폴백까지"**
   - LLM 응답 실패 시 폴백, 무한 루프 방지, 비용 제어 전략
   - genie-cv의 rate limiting + 에러 분류 시스템도 연결

3. **"Agent 워크플로우 모니터링 — LangSmith 기반 관찰 가능성"**
   - fingaroo/kimpro에서의 LangSmith 활용 경험
   - 트레이싱, 비용 추적, 지연 시간 모니터링

#### D. RAG 챗봇 고도화

현재 genie-cv 챗봇 자체가 Agent 역량의 데모:
- **멀티턴 대화 메모리** 추가 → 메모리 시스템 설계 역량 시연
- **Tool Calling 추가** (예: 프로젝트 상세 조회, GitHub 커밋 내역 조회) → Agent 도구 통합 역량 시연
- **평가 체계 적용** → 챗봇 품질 지표를 사이트에 공개

## 4. 이력서 작성 가이드 (토스 권장사항 반영)

토스가 명시한 이력서 가이드에 맞춰:

### "기술 스택보다 문제 정의와 구조 선택 이유 우선"
- ❌ "LangChain, FastAPI, PostgreSQL 사용"
- ✅ "마케팅팀의 캠페인 기획 프로세스가 4단계 수동 반복이었음 → 각 단계를 LangChain 노드로 분리하고, 단계 간 상태를 checkpoint로 영속화하여 장기 실행 가능한 워크플로우로 전환"

### "실험에서 프로덕션까지의 신뢰성 및 확장성 확보 과정"
- MockService 기반 오프라인 평가 → 6가지 Grader로 품질 검증 → 프로덕션 배포 → 모니터링 체계 구축

### "장애/성능/비용 문제 탐지 및 해결 경험"
- LLM 응답 지연 시 타임아웃 + 폴백 전략
- 토큰 비용 관리: 컨텍스트 윈도우 예산 할당
- Wide Event 로깅 + FluentBit → Discord 알림으로 실시간 장애 탐지

### "팀 니즈 발굴부터 플랫폼 기능 연결까지의 과정 및 결과"
- kimpro: 마케팅팀 업무 분석 → 병목 발견 → Agent 자동화 → 사용률/시간 절감 수치
