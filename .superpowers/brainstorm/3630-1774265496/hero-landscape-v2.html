A안 기반, Workflow 안에 AI Agent가 있고 Agent가 전문 도구를 사용하는 구조. 클릭으로 선호 방향을 골라주세요.

A. Workflow 중심 — Agent + Tools 강조

Workflow Service를 큰 박스로 감싸서 Agent 오케스트레이션 구조를 강조. 오른쪽에 AI Tools API가 분리되어 Agent가 전문 도구를 HTTP로 호출하는 관계가 드러남. 하단에 Pub/Sub 이벤트 버스.

B. 좌→우 서비스 흐름 + Agent 확대

Client → Nginx → Services → Workflow(Agent) → Tools 순서로 왼쪽에서 오른쪽으로 읽히는 자연스러운 흐름. Workflow 박스 안에 Agent 오케스트레이션이 크게 자리잡고, Tools API가 오른쪽 상단에 분리. 하단에 Pub/Sub 버스와 DB/Worker 레이어.
